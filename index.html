Recipify App

Welcome to Recipify, the mobile app designed to simplify your cooking experience. Struggling to decide what to cook with the ingredients you have on hand? Recipify is here to help! By suggesting delicious recipes based on the ingredients you already have, Recipify makes meal planning and food choices effortless and enjoyable.

Mobile Part

Our Mobile Application, Recipify is created with Kotlin programming language on Android Studio. It is available to Android devices.

My Recipes Page

User is able to like the recipes offered to him/her and see those recipes in My Recipes page. These liked recipes are stored in SQLite database and stored in users phone. User is also able to remove those recipes.

Home Page

In the home page, user is presented with 6 options of categories. General, Meal, Salads, Drinks, Desserts, Soups. Depending on the category chosen, the user will receive only that type of recipes.

Camera Page

After selecting a category, a camera page opens allowing the user to take a picture of his ingredients. After uploading that photo, the user will receive recipes that correspond to the ingredients in his photo.

All Recipes Page

In this page, user is able to view all the recipes in the system and like the recipes to move them to his My Recipes page. These recipes are fetched from the database.

Backend

When we considered future scenarios of the application on the backend side or because , we wanted it to easily turn into a complex project, we used Spring Boot to easily implement the entity-relation model. We used PostgreSQL as database.

We have a second server to send the image coming from the mobile side to the Yolo model. Here we have another server running on a different port with "Flask".

Artificial Intelligence

Ingredient detection in the captured photos of the users are done by AI. YOLOv8 is used for object detection. The model is trained with multiple ingredient classes and it can perform on images which have multiple ingredients.

More About AI


On the artificial intelligence side of the project, we used yolov8 as the object detection algorithm. The reason we chose this algorithm was that it is an effective, up-to-date detection algorithm with high detection accuracy. We tested yolov8 on 38 classes* that we determined. We created our train, validation and test sets by dividing the number of images of each class in 0.7 ratio for train set, 0.2 ratio for validation set and 0.1 ratio for test set. We made observations by freezing and not freezing the backbone layers of yolo and evaluated in which case the model gave better results, taking into account mAP values per class, precision, recall, F1 and loss metrics. We observed some determined data augmentation parameters and values ​​used in yolo detection processes, which is one of the important parts of training the model, and used them in the code to make the training process more efficient. We ran the yolo model for 50 epochs in the training process. Then, we tested the best model we obtained at the end of the training process on several images which contain multiple ingredients, also were closer to the image format that the user could take, and noted the detection results and times.
